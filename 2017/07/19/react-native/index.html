React-Native环境配置

发表于 2017-07-19 | 分类于 react-native

声明本次react-native环境搭建是基于Ubuntu16.04,同系统不同版本可能有些出入

如需其他系统window,mac请参考中文官方文档

安装

安装react-native前,请自行安装对应的依赖环境

Node
Yarn、React Native的命令行工具
Android Studio

React Native目前需要Android Studio2.0或更高版本。

ANDROID_HOME环境变量

这里由Android Studio自己下载的Android/Sdk目录,S为大写字母

推荐安装的工具

是在不同版本的tslib下执行autogen.sh产生,它们产生的原因一样,是因为没有安装automake工具,
(ubuntu 8.04)用下面的命令安装好就可以了

可选的安装项

Genymotion

比起Android Studio自带的原装模拟器，Genymotion是一个性能更好的选择，但它只对个人用户免费。

  1. 下载和安装Genymotion（genymotion需要依赖VirtualBox虚拟机，下载选项中提供了包含VirtualBox和不包含的选项，请按需选择）。
  2. 打开Genymotion。如果你还没有安装VirtualBox，则此时会提示你安装。
  3. 创建一个新模拟器并启动。
  4. 启动React Native应用后，可以按下F1来打开开发者菜单。

配置Genymotion

  2. 切换到ADB下,选择Usb custom Android SDK tools, 选择你SDK的路径即可
